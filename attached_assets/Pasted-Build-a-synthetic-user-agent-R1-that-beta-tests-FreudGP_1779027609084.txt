No green-checkmark theater. R1 produces raw evidence, not "PASS." Exact text typed, exact text returned, exact network responses, exact screenshots. Judge produces prose critique, not booleans. PASS/FAIL exists only as a filter for failures.md — never as a substitute for evidence.
The HTML report must be reviewable in under 30 minutes. Every interaction shows step + R1's input verbatim + app's response body + screenshots + judge critique. No collapsed sections, no "Click to expand."
The live view must show R1's actual inputs and the app's actual outputs as they happen — not "Step 4 of N in progress." Spec below.
Network traffic captured raw. Every /api/* request and response body in network.log as JSONL.
Three screenshots per interactive step (before / after typing / after app responds). Navigation-only steps get one. The sanity check distinguishes the two cases.
If R1 cannot complete a step, the failure is logged with the actual error, page HTML at moment of failure, and a screenshot. No skipping and marking green.
Per-step API-call expectations are route-specific. "Ask a thinker" expects POST /api/ask specifically. Boolean checks forbidden.
Critical button clicks wrapped in Promise.all([page.waitForResponse(predicate), click]) so R1 doesn't silently advance when a click fails to fire.

What's different about this app — FreudGPT's defining feature
FreudGPT is a database-grounded philosophical consultation app. Its entire premise is that responses are traceable to actual retrieved positions from a corpus — not freely generated. The Archive panel on the right shows exactly which positions were retrieved for each query. The whole product collapses if grounding fails.
R1's heaviest job is enforcing grounding. The five invariants that elevate to CRITICAL VIOLATION status:
Invariant A — Every response is grounded in actually-retrieved positions. For every /api/ask call, the SSE stream emits a retrieval event before any tokens, containing the position IDs retrieved. R1 captures these IDs. Then R1 sends the full streamed response + the retrieved position texts to the judge with: "Does every major claim in the response trace to one of these positions? List any claim that doesn't have a clear basis." If the judge identifies ungrounded claims (i.e., the LLM invented content the corpus doesn't support), that's a CRITICAL VIOLATION. The blueprint says the prompt forces grounding — if grounding fails, the central feature is broken.
Invariant B — The low-relevance warning fires when it should. Per blueprint, when max_similarity < 0.25 the LLM must explicitly acknowledge the retrieved positions don't address the question. R1 deliberately asks an off-topic question (e.g., ask Freud about cryptocurrency mining) and verifies: (a) the retrieval event shows low similarity scores, (b) the response explicitly acknowledges the mismatch rather than pretending to answer with hallucinated material. If the response barrels ahead and invents Freud-on-Bitcoin content without acknowledging the gap, CRITICAL VIOLATION.
Invariant C — Database integrity per the diagnostic. The 20-check diagnostic verifies per-thinker counts and schema. R1 runs it first and last. Any count drift (e.g., Freud was 19,077 before R1's run and is 19,072 after) means R1's testing accidentally mutated the corpus. CRITICAL VIOLATION.
Invariant D — Memory Mode tree growth is bounded and tagged. Per blueprint, after each Memory Mode exchange Claude updates a Wittgenstein-numbered tree with nodes tagged ASSERTS/REJECTS/ASSUMES/OPEN/RESOLVED. R1 sends a Memory Mode exchange and verifies: tree grew by ≥ 1 node, every new node value starts with one of the five tags, every new node ID follows Wittgenstein decimal format (1.0, 1.1.2, etc.). If a node lacks a valid tag or has malformed numbering, CRITICAL VIOLATION.
Invariant E — Longform doesn't duplicate sections. The blueprint emphasizes that the coherence skeleton specifically prevents the "many small essays cobbled together" failure mode via must_not_repeat lists and per-section unique roles. R1 generates a small longform document and verifies: (a) no two sections have identical or near-identical opening sentences, (b) the state's claims_made array grows across sections (deduplication works), (c) section roles in the skeleton are distinct (no two sections with role="intro" etc.). If sections repeat content, CRITICAL VIOLATION — the central NeuroText-style claim of this app is false.
Architecture

Browser automation: Playwright (Node, single-file tools/r1/run.mjs). Standalone Node project, not in any workspace.
R1's brain: Anthropic API. Model: claude-opus-4-7. Key in ANTHROPIC_API_KEY.
Judge: Separate Claude API call per interaction. Prose critique. For grounding checks (Invariants A and B), the judge gets both the response and the actual retrieved position texts.
Live view: node:http server on port 7777. Zero extra deps.
Target app: FreudGPT running locally. Flask on port 5000, longform TS service on port 3001. Default APP_URL=http://localhost:5000. Auth deferred per blueprint — diagnostic is unauthenticated.

What R1 is testing — the functions
Function 1 — System Diagnostic (FIRST, MANDATORY)

Capture baseline: query the diagnostic endpoint for current per-thinker counts.
Click 🩺 DIAGNOSTIC button or navigate to the diagnostic modal.
Click "Run Full Diagnostic." Expected route: POST /api/diagnostic/run.
Capture full JSON response with all 20 check results.
Save as diagnostic-before.json.
Per-thinker counts must match blueprint (freud 19,077 / kuczynski 17,499 / jung 2,910 / nietzsche 2,838 / bergler 1,924 / hume 1,114). Any deviation → judge concern with the exact diff.
If any AI provider check fails, log; if DATABASE_URL or embedding service fails, R1 aborts dependent tests.
If longform service health check fails, R1 skips Function 7 (longform).

Function 2 — Thinker selection + Knowledge panel

Verify all 6 thinkers visible (Freud, ZHI/Kuczynski, Jung in primary row; Hume, Nietzsche, Bergler in Related header).
Click each thinker avatar.
Expected route: GET /api/topics/<thinker> (per blueprint).
Verify count badges match blueprint counts.
Verify knowledge panel popup opens with starter questions.
Verify active-thinker pill updates.

Function 3 — Single consultation (happy path, Invariant A)
For each primary thinker (Freud, Kuczynski, Jung):

Select thinker. Default provider ZHI 4 (DeepSeek). Standard mode.
Type a question well within the thinker's wheelhouse (Freud: "What is the role of repression in neurosis?"; Kuczynski: a question matching his published work; Jung: "How does the collective unconscious differ from the personal unconscious?").
Click Consult. Expected route: POST /api/ask (SSE).
Capture all SSE events: retrieval, token (many), sources, done.
From retrieval event: capture positions_scanned, positions_above_threshold, max_similarity, mean_similarity, top_k, retrieved_position_ids, domains_covered.
Capture full assembled response text.
Verify Archive panel displays the actually retrieved positions.
Verify Invariant A: judge receives the full response + the retrieved position texts (fetched via GET /api/positions/search?thinker=...&q=... if needed, or extracted from the sources event) and reports per-major-claim grounding. Any unsupported invention → CRITICAL VIOLATION.

Function 4 — Invariant B: low-relevance warning

Select Freud.
Ask: "What are the optimal asset allocations for a 2030 retirement portfolio in a high-inflation environment?"
Capture retrieval event — max_similarity should be very low (< 0.25).
Capture response.
Verify Invariant B: response must explicitly acknowledge the corpus doesn't address this question. Specific phrases to look for (judge decides): "I don't have positions on that," "my work doesn't address," "the retrieved material is tangential," etc.
If response invents Freudian commentary on retirement investing as if it were grounded, CRITICAL VIOLATION.

Function 5 — Provider switching
For each of the 5 providers (ZHI 1 Anthropic, ZHI 2 OpenAI, ZHI 3 Grok, ZHI 4 DeepSeek, ZHI 5 Perplexity — skip any not configured per the /api/providers response):

Ask the same question (e.g., "What is the unconscious?") with Freud selected.
Verify response streams.
Verify retrieval event content is structurally similar across providers (same retrieval logic; providers should retrieve the same positions for the same query).
Judge concern (not violation): if two providers produce byte-identical responses, one fell back to the other.

Function 6 — Memory Mode (Invariant D)

Create a project "R1 Test Project" via POST /api/memory/projects.
Capture project_id.
Create a session via POST /api/memory/projects/:id/sessions.
Toggle Memory Mode ON.
Send first message: a substantive question that should generate ASSERTS-type nodes.
Expected route: POST /api/memory/ask.
After response, fetch GET /api/memory/projects/:id/tractatus. Capture tree node count.
Send second message that should generate REJECTS or RESOLVED nodes (e.g., contradict something the thinker said earlier in the conversation).
Re-fetch tractatus. Verify Invariant D: new nodes have valid tags and Wittgenstein decimal IDs. Tree grew. If not, CRITICAL VIOLATION.
Send third message introducing a distinctive fact ("the test code is XQ-77-blue, remember this").
Verify tree includes a node for this fact.
Test cross-session persistence: create a new session in the same project, ask "what was the test code?" — response must contain XQ-77-blue. If it doesn't, CRITICAL VIOLATION (memory injection broken).
Click 🌳 Tree button. Verify viewer opens. Verify download-as-JSON works (capture downloaded file).
Test compression edge: if there's a way to seed a project past 500 nodes without burning massive tokens, do it; otherwise skip with a logged reason.

Hash the first 100 characters of each section. No two hashes identical. If any are, CRITICAL VIOLATION (duplicate sections).
Compute pairwise text similarity between sections using simple shingle-overlap. If any pair > 70% similarity, CRITICAL VIOLATION.
Verify the claims_made array in state-after snapshots grows monotonically across sections (no shrinking, no resets).
Verify section roles in skeleton are distinct (no duplicate role values).
